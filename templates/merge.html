Merge Sort Explaination

Merge sort is a variation of the sorting algorithm. It takes a list/array and spilts it in half and keeps splitting it in half until it is down to single elements. Once the elements are on their own the algorithm then starts comparing the elements. It starts by comparing two single elements together and then it compares two pairs, so on and so forth. When merge sort is comparing more than one element it will look at the element at index 0 in both of the arrays until it empties the arrays back into the original one and the array is sorted.


When comparing merge sort to other sorting algorithms it performs well on any size of array where as some other variations of sorting e.g quick sort only work well on smaller arrays. Merge sort has a good speed of execution being a complexity of O(n log n). This means that the bigger n is then the more time the algorithm will take, however it is increasing at a smaller rate than n^2. For example, if you had an array of 9 elements then it would take merge sort 4 steps to sort the array. The same goes for arrays with 10, 11, 12, 13, 14, 15 and 16 elements as these also only take 4 steps to merge sort.

One of the trade offs of merge sort is that it takes up memory when sorting the array as it takes the elements out of the array in order to sort them. Where as other sorting options may be slower but take up less memory.
